# Security notes

## Canonical domain

The canonical production domain is:

```text
https://www.advocaciabarreto.com
```

Requests to `https://advocaciabarreto.com/*` are redirected permanently to the same path on `https://www.advocaciabarreto.com/*` by `vercel.json`.

Known React Router pages are emitted as static route HTML during `npm run build` so their initial canonical and `og:url` tags match the final `www` URL before client-side React runs.

## Security headers

Security headers are configured centrally in `vercel.json`:

```text
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
X-Frame-Options: DENY
Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=(), usb=(), accelerometer=(), gyroscope=(), magnetometer=()
X-Permitted-Cross-Domain-Policies: none
Access-Control-Allow-Origin: https://www.advocaciabarreto.com
Cross-Origin-Opener-Policy: same-origin
```

`Cross-Origin-Embedder-Policy` and `Cross-Origin-Resource-Policy` are intentionally not enabled yet because the site uses third-party maps, fonts, analytics scripts, and external images.

## CSP report-only

The initial policy is observation-only and is not enforcement:

```text
default-src 'self'; base-uri 'self'; form-action 'self'; frame-ancestors 'none'; object-src 'none'; img-src 'self' https: data: blob:; style-src 'self' 'unsafe-inline' https:; font-src 'self' https: data:; script-src 'self' 'unsafe-inline' 'unsafe-eval' https:; connect-src 'self' https:; frame-src 'self' https:; upgrade-insecure-requests; report-uri /api/csp-report
```

CSP violation reports are accepted at `/api/csp-report` and logged in Vercel function logs with a small sanitized field set.

https://www.googletagmanager.com
https://www.google-analytics.com

https://www.google.com/maps
https://api.whatsapp.com
https://maps.app.goo.gl
https://picsum.photos
https://fastly.picsum.photos

SRI was not added in this pass because the external scripts are loaded dynamically by existing analytics code and the Google Fonts CSS response is generated by Google. Revisit SRI only for static third-party assets with stable versioned URLs.

## Phase 2

- Review CSP reports and replace the broad `https:` allowances with explicit host allowlists.
- Move from CSP report-only to CSP enforcement after reports are clean.
- Replace `'unsafe-inline'` and `'unsafe-eval'` with nonce/hash-based allowances where feasible.
- Evaluate GTM and Meta Pixel separately if marketing tracking is approved for a later phase.
- Consider Sentry or another application monitoring tool after the security baseline is stable.
